### Track what you eat, in the words you'd say it.

A privacy-first, AI-assisted calorie & macro tracker built for how India actually eats —
katoris and rotis, not cups and ounces.

## What it does

Type a meal the way you'd say it to a friend — English, Hindi, or Hinglish:

> **"2 roti aur ek katori dal, 5 eggs"**

Aahaar's AI extracts *what* you ate (`roti ×2 · dal ×1 katori · egg ×5`), and a **local nutrition
database of 287 Indian foods** computes the calories and macros — on your phone. The AI never
invents a single calorie number.

## Features

- 🗣️ **Natural-language logging** — Hinglish included; quantities and desi units (katori, roti, glass) understood
- 🔒 **Private by design** — all data lives in on-device SQLite; no account, no cloud database, no tracking
- 🍛 **287-food Indian nutrition DB** — from masala dosa to misal pav, with realistic portion weights
- ⭐ **Saved meals** — your usual dal-chawal re-logged in one tap, no AI call
- 📊 **Goals & progress** — Mifflin–St Jeor targets, macro bars, streaks, days-within-goal
- 📅 **Calendar & trends** — month at a glance, rolling averages, tap any day for its meals
- 🎨 **Theming** — light/dark + five accent colours, all persisted
- 📦 **Account-free backup** — export/import a single JSON file via your own Drive/iCloud/Files
- ✈️ **Offline mode** — a built-in parser works with zero connectivity

## Privacy architecture

Only the meal *text* ever leaves the device — never your identity, and never your history.

```
meal text ──► parse cache (repeat meals never hit the network)
          ──► Cloudflare Worker proxy ──► LLM        (extraction only: {food, qty, unit})
          ──► offline heuristic parser               (no network at all)

numbers   ◄── local SQLite nutrition DB              (calories & macros computed on-device)
```

- The LLM returns **words, not numbers** — nutrition math happens against the local database
- The API key lives in a Cloudflare Worker secret, never in the app bundle
- Repeated meals are served from an on-device parse cache
- AI-estimated foods (DB misses) are validated: density clamps + 4-4-9 macro reconciliation

## Running locally

```bash
cd mobile
npm install
npx expo start          # Expo Go; press `a` for the Android emulator
```

- **AI parsing (optional):** copy `mobile/.env.example` → `mobile/.env` and set either a direct
  provider key (dev only) or `EXPO_PUBLIC_PARSER_URL` to your deployed worker. With no key at all,
  the offline parser takes over.
- **Worker:** `cd worker && npx wrangler secret put GROQ_API_KEY && npx wrangler deploy`
  (see `worker/README.md`).
- **Native build** (icon/splash need one): `npx expo run:android`.
- **Engine tests:** `cd mobile && node scripts/test-matcher.mjs`.
- **Regenerate brand assets:** `cd brand && npm install && node generate-icons.mjs`.
